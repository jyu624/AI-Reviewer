## 🎖️ 评审流程设计

### Phase 1: 初赛阶段 (快速评审)
```
参赛项目提交 → AI快速预审 → 基础筛选 → 进入复赛
     ↓              ↓              ↓          ↓
   源码上传      10秒评分      70分以上    20个项目
```

### Phase 2: 复赛阶段 (详细评审)
```
复赛项目 → AI详细评审 → 专家复审 → 晋级决赛
    ↓            ↓              ↓          ↓
 20个项目     30秒深度分析    人工确认    5个项目
```

### Phase 3: 决赛阶段 (专家评审)
```
决赛项目 → AI专家评审 → 评审团终审 → 获奖公布
    ↓            ↓                ↓          ↓
  5个项目     60秒专家分析      综合评分    获奖名单
```

## 🏅 评分标准体系

### 优秀项目 (90-100分)
- 技术架构设计优秀，代码质量上乘
- 功能完整，创新性突出
- 商业价值高，市场前景良好
- 测试覆盖完善，文档齐全

### 良好项目 (75-89分)
- 技术实现扎实，架构合理
- 功能较为完整，有一定创新
- 商业价值不错，有发展潜力
- 代码质量良好，基本规范

### 及格项目 (60-74分)
- 技术实现基本完成，架构清晰
- 核心功能实现，有改进空间
- 商业价值一般，需进一步验证
- 代码质量及格，有规范

### 待改进项目 (<60分)
- 技术实现存在问题，架构不合理
- 功能不完整，体验差
- 商业价值不明，创新不足
- 缺乏测试，代码质量低

## 🚀 部署和使用

### 快速部署
```bash
# Linux/macOS
chmod +x deploy-hackathon.sh
./deploy-hackathon.sh

# Windows
deploy-hackathon.bat
```

### 基本使用
```bash
# 评审单个项目
java -jar hackathon-reviewer-2.0.jar review /path/to/project QUICK

# 批量评审项目
java -jar hackathon-reviewer-2.0.jar batch project1 project2 DETAILED

# 查看排行榜
java -jar hackathon-reviewer-2.0.jar leaderboard

# 查看统计信息
java -jar hackathon-reviewer-2.0.jar stats
```
